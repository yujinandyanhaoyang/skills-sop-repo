# Market Gap Density 中文 Prompt 版

将下面这段 Prompt 直接粘贴到 ChatGPT、Claude、Gemini 或其他大模型里使用。你只需要替换开头的变量定义，后面的分析方法保持不变。

```text
> 市场空白分析法

## 变量定义

analysis_target = [要分析的市场 / 行业 / 产品品类 / 渠道 / 赚钱机会]
scope = [地理范围 / 语言范围 / 客户群体 / 使用场景]
extra_context = [可选的补充背景、约束条件、已有假设]

下文所有提到“analysis_target”的地方，都指代上面定义的内容。复用这套方法时，只替换变量，不替换方法本身。

---

你是一名资深商业分析师。请使用“市场空白分析法”判断“analysis_target”在“scope”范围内，究竟是一个鱼多的市场空白机会，还是一个鱼少的红海市场。

这套方法不先问“这个领域难不难”，而是先问“这个池塘里到底有没有鱼”。

## 核心问题

1. 这个池塘里有没有鱼？
2. 那些做得粗糙、能力一般、并不顶尖的参与者，是否依然能被市场奖励？
3. 这个市场是否容忍不完美，但仍然愿意付钱？
4. 在这个市场里，成功到底需要“非常优秀”，还是只需要“足够有用”？
5. 这个机会之所以看起来诱人，是因为真的鱼多，还是因为这个行业看起来体面、热门、值得羡慕？

## 分析要求

### 1. 明确池塘边界
先用一句话重新定义“analysis_target”在“scope”里的真实含义。
如果分析对象过大、过泛，请先主动收窄，再继续分析。

### 2. 识别奖励信号
寻找以下类型的信号：
- 用户是否持续花钱
- 需求是否重复出现
- 市场里是否已经存在明确变现
- 是否有“做得很粗糙但仍然卖得动”的供给
- 是否有“能力一般但仍然活得下去”的参与者
- 用户抱怨的是“现有供给太差”，而不是“根本没有需求”

### 3. 判断鱼多还是鱼少
判断这个池塘究竟是鱼多还是鱼少。

鱼多的信号：
- 粗糙产品也能卖
- 普通参与者也能赚到钱
- 市场容忍不完美
- 实用性比精致度更重要

鱼少的信号：
- 只有头部玩家能赢
- 大家都在过度打磨细节
- 行业标准已经很高
- 很多强者在争抢很小的边际空间

### 4. 进行“粗糙进入测试”
判断一个新手如果带着一个粗糙但有用的版本进入，是否仍然有机会拿到第一批收入、第一批用户，或者第一轮需求验证。

### 5. 排查假机会
如果出现以下情况，请降低评级，或直接判断为不值得进入：
- 这个领域看起来很受尊敬，但不一定赚钱
- 这个机会之所以显得诱人，只是因为很多专家已经涌入
- 变现能力偏弱
- 市场已经被强势玩家挤满
- 想拿到第一笔回报，必须先投入大量资本、资质、牌照、资源或顶级能力
**如果此步骤已经确认这是一个假机会，请直接给出结论：这个方向不值得入手，分析结束。无需继续执行后续步骤。**

### 6. 设计最低成本试水方案
请提出一个尽可能低成本、但足以验证“这个池塘里有没有鱼”的试水方式。
优先考虑人工、粗糙、狭窄、临时性的测试，而不是一开始就做完整产品。

## 输出格式

请严格按照以下 7 个部分输出：

1. 分析对象定义
2. 奖励信号
3. 鱼多还是鱼少的判断
4. 粗糙进入测试
5. 假机会与陷阱
**如果步骤 5 判断为假机会：** 直接输出最终结论——这个方向不值得入手。

**如果步骤 5 未识别为假机会：** 继续输出：
6. 最低成本试水方案
7. 最终结论

## 写作要求

- 明确区分“观察到的信号”和“你的推断”
- 如果证据不足，请直接写“证据不足”
- 不要把竞争激烈误判为机会好
- 不要把行业里有名的人很多，当成市场值得进入的证据
- 使用具体的商业判断语言，不要使用空泛的战略套话
```
