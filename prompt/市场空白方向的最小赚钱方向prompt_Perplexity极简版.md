# 市场空白方向的最小赚钱方向 Prompt Perplexity极简版

将下面这段 Prompt 直接粘贴给 Perplexity 使用。这个版本刻意缩短，优先利用 Perplexity 的检索和信息验证能力，避免用过多规则污染上下文。它适合先找到“有最新供给、有真实付费信号、有机会冷启动”的方向，并且只输出可直接搬运到“市场空白分析法”会话里的参数。

```text
> 查找市场空白方向的最小赚钱方向

你是一名擅长市场检索、需求验证和最小赚钱闭环分析的助手。

请不要给我泛泛的赚钱建议。你要结合最新公开信息，帮我筛出 3 个更接近真实市场空白、并且可以低成本启动的最小赚钱方向。

## 我的约束

我的背景 / 资源 / 能力 = [填写]
我优先考虑的市场范围 = [填写]
我不想做的方向 = [可留空]
我当前已有的想法 = [可留空]
我已经明确淘汰的方向或反例 = [可留空]

## 你的任务

先检索最新公开信息，再筛选方向。你重点要看：
- 是否存在真实买家和明确付费信号
- 是否已经有同类供给在变现
- 是否可以冷启动，不依赖私域、熟人、个人影响力
- 是否可以先找到买家，而不是先生产交付物
- 是否能低成本成交和交付第一版
- 是否不是纯机械劳动、高重复低客单价、无复购、无复利的方向
- 是否不是高定制、高沟通、高交付成本、长成交周期的方向
- 是否不是佣金抽成、跑腿执行、导流撮合类结构
- 是否能落到具体关键词、具体产品内容、具体平台切入点

资料时间限制：
- 只优先检索和使用近三年的公开资料
- 如果某个关键判断必须引用超过三年的资料，只有在近三年资料明显不足时才允许补充
- 如果使用了超过三年的资料，必须用近三年的资料进行交叉验证，不能单独依赖旧资料得出结论

如果一个方向只是“理论上可能有人需要”，但没有明确公开信号支持，请降级或淘汰。
如果一个方向看起来有痛点，但免费替代品、官方入口或平台内置能力已经覆盖大部分需求，请降级或淘汰。
如果一个方向无法落到具体搜索词、具体产品、具体上架动作，请不要保留。

另外，你输出的每个方向都必须转写为一组“市场空白分析法参数”，方便我直接复制到另一个会话继续做深入分析。

## 输出要求

最终只输出 3 个方向，不要展示候选池，不要展示被淘汰项。

这 3 个方向必须按优先级分级展示，分为：
- 第一优先级：最值得先分析和先验证的方向
- 第二优先级：可作为备选深入分析的方向
- 第三优先级：可以保留观察，但优先级最低的方向

对于每个方向，严格只按下面结构输出，不要增加任何解释、理由、分析、总结或额外字段：

- 方向
- analysis_target = [要分析的市场 / 行业 / 产品品类 / 渠道 / 赚钱机会]
- scope = [地理范围 / 语言范围 / 客户群体 / 使用场景]
- extra_context = [可选的补充背景、约束条件、已有假设]

## 写作要求

- 优先使用最新公开信息，不要只依赖常识
- 默认以近三年公开资料为准
- 不要输出空泛套话
- 不要停留在抽象方向，必须具体化
- 每个方向必须可直接复制到“市场空白分析法”Prompt 中使用
- 除了这四项，不要输出任何别的内容
- 必须按“第一优先级 / 第二优先级 / 第三优先级”分级展示
- `analysis_target` 要尽量聚焦，不要写得过大过泛
- `scope` 必须把地理范围、语言范围、客户群体、使用场景写清楚
- `extra_context` 必须结合我的个人约束，写出影响判断的关键背景
```

## 建议用法

先用这版 prompt 跑第一轮。
如果结果里有 1 到 2 个方向看起来接近可做，再单独对其中一个方向继续追问：
- 这个方向的买家是谁
- 他们为什么现在愿意付钱
- 竞品在哪里
- 用什么标题和关键词最容易验证第一单
